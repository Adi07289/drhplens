# Phase 4 — UI Design Contract: Peer Comparator + Read-Only GMP + Indian Formatting & Glossary

> Visual and interaction contract for Phase 4's three NEW user-facing surfaces — the peer-multiples comparison table, the de-emphasised read-only GMP display, and the app-wide Indian-number formatting + glossary tooltips — generated by gsd-ui-researcher; consumed by gsd-planner, gsd-executor, gsd-ui-checker, gsd-ui-auditor.
>
> **This contract EXTENDS the Phase 1/2/3 design system. It restyles nothing.** Every `--drhp-*` token, the four-size / two-weight type scale, the 60/30/10 colour split, the citation-chip + expander renderers, the `.drhp-not-disclosed` honesty note, the disclaimer surfaces, the four mobile breakpoints, the 44×44 touch-target rule, and the **no red/green, no badges** invariant are INHERITED VERBATIM. This document specifies only the deltas Phase 4 adds and how they reuse inherited primitives. Where a value is unchanged, this contract states "inherits Phase N §X" rather than re-deriving it.

## Scope of This Contract

Phase 4 ships **three NEW user-visible surfaces**, all rendered on the existing snapshot page (`pages/02_snapshot.py`), plus one backend artifact that is **not** user-visible this phase:

1. **Peer-multiples comparison table** (PEER-01, PEER-02) — the DRHP's own "Comparison with Listed Peers" set, cited back to its DRHP section, with P/E, P/B, EV/EBITDA, ROE per company, each cell carrying a per-cell provenance marker.
2. **Read-only GMP display** (GMP-01, GMP-02) — a deliberately de-emphasised, monochrome, spread-across-aggregators grey-market-premium block, low on the page, behind a "What is GMP? Why we don't trust it" disclosure, with a persistent caveat. Computationally isolated from any model pipeline.
3. **Indian-context formatting + glossary tooltips** (UI-04) — one shared `format_inr` utility (₹ + Indian digit grouping + auto-scaled lakh↔crore + `tabular-nums`) applied app-wide, plus CSS-only glossary tooltips on the core Indian-IPO vocabulary.

**Not user-visible this phase:** the survivorship-corrected historical IPO dataset (backend CSV/parquet, foundation for Phase 5). It has **no UI surface in Phase 4** and is therefore out of scope for this contract, except that its ~7%-median sanity-check divergence flag, if it fires, surfaces on the existing `/methodology` page as plain text (no new visual).

Everything else on the snapshot page (catalogue cards, metadata header, red-flag table, IDF risk list, the six Phase 2 blocks, methodology pane, Q&A chat, citation chips, disclaimer surfaces) is **already shipped and unchanged**.

## Discretion Resolved (the four genuinely-open questions)

`04-CONTEXT.md` explicitly deferred four visual/interaction decisions to this UI-SPEC. They are resolved here, prescriptively.

### R-1 — Glossary-tooltip interaction & mechanism: **CSS-only `:hover` / `:focus-within` popover on an inline term trigger. No JavaScript.**

Streamlit renders our markdown as static HTML with no JS hook, so the mechanism must be pure CSS/HTML injected through the existing `unsafe_allow_html` path.

- **Trigger:** the glossary term is wrapped in an inline `<span class="drhp-glossary" tabindex="0" role="button" aria-describedby="…">RPT</span>` with a **dotted-underline** affordance (`text-decoration: underline dotted`, `cursor: help`). The dotted underline is the "there is more here" signal — deliberately distinct from the accent citation chip (filled superscript pill) so the two systems never read as the same thing.
- **Reveal:** a sibling `<span class="drhp-glossary-pop" role="tooltip">` shown on `:hover`, `:focus`, and `:focus-within` of the trigger. Popover is `surface-secondary` background, `border-subtle` 1px, `8px` radius, Small 12px text, max-width 260px, positioned above/below with CSS only.
- **Desktop:** hover reveals; keyboard `Tab` focus reveals (accessible).
- **Mobile:** **tap** the term → it receives focus → popover shows; tap elsewhere dismisses (focus lost). The trigger carries the inherited **44×44px minimum tap target** (transparent `::before` enlarger, same mechanism as `.drhp-cite::before`).
- **Accessibility:** the popover text is linked via `aria-describedby` so screen readers announce the definition inline; `prefers-reduced-motion` disables any fade (inherited rule). No `title=` attribute is used as the primary mechanism (title tooltips are hover-only, non-keyboard, non-mobile).
- **Why not `st.help` / `help=`:** those attach only to Streamlit widgets, not to inline acronyms inside rendered prose or table cells. Unusable for in-text glossary terms. Rejected.

### R-2 — Peer-table responsive layout at 375px: **a horizontal-scroll table with rows = companies, columns = the four multiples, sticky company-name left column. NOT stacked per-company cards.**

**Axis orientation (locked here): rows = companies (the IPO first, then each DRHP-named peer); columns = `P/E · P/B · EV/EBITDA · ROE`.** This matches how Indian DRHP "Comparison with Listed Peers" tables are actually laid out (companies down, ratios across), so it anchors cleanly to PEER-01. It also scales the right way: **adding peers grows the table vertically (natural scroll), never horizontally** — the column count is fixed at 5 (name + 4 metrics) regardless of peer count.

- **Mechanism:** reuse the proven Phase 2 financials pattern verbatim — `overflow-x: auto` wrapper + `position: sticky; left: 0` on the company-name column. At 375px the four metric columns scroll horizontally if needed while the company name stays pinned, so the user always knows whose row they are reading.
- **Why NOT stacked cards on mobile:** a peer comparison's entire purpose is the **column-wise scan** — reading every company's P/E *down one column* to see where this IPO sits. Per-company cards isolate each company and **destroy** that cross-peer comparison. The table preserves it; cards defeat the feature. Rejected.
- **Why NOT columns = companies:** would make column count grow with peer count (6 peers → 7 columns), forcing deep horizontal scroll that buries peers off-screen — the exact failure mode R-1 of Phase 3 warned about. Rejected.

### R-3 — Per-cell provenance-flag visual: **a small superscript MUTED single-letter source key after the value, plus a persistent legend below the table. Distinct from the accent citation chip.**

- Each cell value is followed by a `<sup class="drhp-provenance-flag">s</sup>` — a lowercase letter, **muted grey (`text-muted`), unfilled, ~10px**. Because it is muted and unfilled, it is visually unmistakable from the **accent, filled** citation chips — one grammar for "market-data source of this number," a different grammar for "DRHP citation of the peer set."
- **Legend** (persistent, Small 12px muted, directly below the table): `Sources: d DRHP · s screener.in · y Yahoo Finance · n NSE / BSE`. Lightweight — a one-line legend, **not** a footnote apparatus.
- Each provenance letter also carries an `aria-label` / the glossary-popover mechanism (R-1) revealing the full source name on hover/tap, so the marker is self-describing without forcing a legend lookup.
- **No colour** encodes provenance (monochrome invariant). A DRHP-sourced cell and a screener-sourced cell differ only by the muted letter, never by fill or hue.

### R-4 — GMP collapsed/expanded visual: **a monochrome, deliberately-quiet block, LAST in the read section, with a persistent one-line caveat, a monochrome multi-source range strip, and the explainer behind a `st.expander`.**

The GMP block is the **quietest surface in the app** by design — it uses **no accent colour at all** (accent is the "the system points here" colour; we explicitly do *not* point at GMP). Structure, top to bottom:

1. **Heading** (`.drhp-snapshot-block-heading`, 20/600, but rendered calm): `Grey-market premium (unofficial)`.
2. **Persistent caveat** (always visible, Small 12px muted, immediately under the heading): the one-line provenance/reliability warning — never collapsed.
3. **The spread visualisation** — a monochrome **range strip**: a `surface-secondary` horizontal track with one **muted tick per aggregator** placed by value, and the min–max range printed as the headline (`₹38–₹52 across 3 sources`). **The divergence between ticks is the visual proof that GMP is unreliable** (D4-01). No accent, no red/green, no single authoritative number.
4. **Per-source list** (Small 12px muted): `Aggregator A ₹38 · Aggregator B ₹47 · Aggregator C ₹52 · as of {date}`.
5. **The disclosure** — an inherited `st.expander`, header copy `What is GMP? Why we don't trust it`, default-collapsed, containing the genuine explanation (unofficial, unregulated, manipulable, no provenance guarantee, excluded from every forecast).

The whole block sits **below all Phase 2/3 read blocks, immediately above the Q&A `2xl` divider** (low on the page, D4-02). It reads only from the `data/gmp/<drhp_id>.json` cache (D4-03) and imports nothing from any model/forecast module.

## Typography

Inherited four-size, two-weight scale (`Small 12 / Body 16 / Heading 20 / Display 28`; weights 400 + 600). No new sizes, no new weights.

| Role | Size | Weight | Line Height | Phase 4 usage |
|------|------|--------|-------------|---------------|
| Small | 12px | 400 | 1.5 | Provenance legend; provenance superscript (~10px, an inherited sub-Small exception already used by the 0.7em citation chip); GMP persistent caveat; GMP per-source list; GMP "as of" date; glossary popover text; peer-table column headers; the `NM` / `—` cell notes |
| Body | 16px | 400 / 600 | 1.55 | Peer-table cell values (400, `tabular-nums`); company-name column (600 for the IPO's own row label `This IPO`, 400 for peer names); GMP range headline `₹38–₹52 across 3 sources`; glossary term trigger inline in prose |
| Heading | 20px | 600 | 1.3 | Block headings `Comparison with listed peers` and `Grey-market premium (unofficial)` (reuse `.drhp-snapshot-block-heading`) |

**Rules:**
- Peer multiples are ratios, not rupee amounts: render as a plain number to **one decimal** with `tabular-nums`, no ₹ (e.g. `24.3`). ROE renders as a percentage to one decimal (`18.6%`).
- Rupee magnitudes (issue size, GMP ₹ value, any ₹ cell) render through `format_inr` (see §Indian Formatting) with `tabular-nums`.
- The GMP block uses no bold emphasis on the value — it is deliberately typographically calm (no Display size, no 600 headline number) so it never visually competes with the peer table or the red-flag surface.
- The provenance superscript reuses the citation-chip's sub-Small sizing convention (`~0.7em`) but in muted grey and unfilled — it is a source key, not a new type role.

## Visuals — Peer-Multiples Comparison Table Contract (PEER-01, PEER-02)

Renders inside a block matching `.drhp-snapshot-block` chrome (white bg, `border-subtle`, `8px` radius, `lg` padding), wrapped with `st.container(border=True)` (never a split div — Streamlit lesson). Recommended position: **directly after the Phase 2 Key Financials block** (topically adjacent — "here are its financials, here is how it compares"). Planner picks the exact ordinal within the locked P2/P3 order.

**Structure (new class `.drhp-peer-table`):**
- **Block heading** (reuse `.drhp-snapshot-block-heading`): `Comparison with listed peers`.
- **Peer-set citation** (inherited accent `.drhp-cite` chip on the heading/caption): anchors the whole peer SET to the DRHP's "Comparison with Listed Peers" section, with the inherited `st.expander` revealing the verbatim DRHP span + `View DRHP page N →` (D4-04). **This is the only accent element in the block.**
- **The table** (`.drhp-peer-table`, wrapped in the inherited `.drhp-fin-table-wrap` `overflow-x:auto`):
  - **Rows = companies.** Row 1 = **the IPO itself**, name cell reads the issuer name + a muted `This IPO` tag, row carries a neutral `surface-secondary` fill for orientation. Rows 2…N = each DRHP-named peer, name only.
  - **Columns = `Company · P/E · P/B · EV/EBITDA · ROE`.** Company column is `position: sticky; left: 0`. Metric columns right-aligned, `tabular-nums`.
  - **Cells:** each value followed by its **provenance superscript** (R-3). IPO-row cells that are DRHP-derived carry `ᵈ`; peer cells carry `ˢ`/`ʸ`/`ⁿ` per source.
- **Provenance legend** (`.drhp-provenance-legend`, Small muted, below the table): `Sources: d DRHP · s screener.in · y Yahoo Finance · n NSE / BSE`.
- **Per-block disclaimer** (inherited `PER_ANSWER_FOOTER`: `Informational only — not advice.`).

**Cell edge cases (Claude's-discretion number formatting, following Phase 2/3 conventions):**
- **Missing from every source:** `—` (em dash), `aria-label="Not available from any source"`. Never a zero, never a fabricated or interpolated value (D4-05, mirrors P2-L8).
- **Negative / undefined P/E (loss-making company):** render `NM` (Small muted) with the glossary-popover note `Not meaningful — the company reported a loss`. Never a red number, never a fabricated positive. (A negative P/E is genuinely not interpretable; showing `NM` is the honest choice.)
- **Negative values elsewhere (e.g. negative ROE):** parentheses in the **same** `text-primary` colour — `(4.2%)` — the inherited financials no-red-for-losses rule.

**Honest empty-state (D4-06):** when the DRHP names no listed peers, the block renders its heading + the `.drhp-not-disclosed` italic note **`This DRHP disclosed no listed-peer comparison.`** — no table, no fabricated peer set, no sector-peer substitution (that fallback is deferred).

**What NOT to do (anti-patterns):**
- No red/green on any multiple; a low and a high P/E are visually identical (D4-09).
- No "cheap" / "expensive" / "overvalued" / "undervalued" label anywhere (also banned tokens).
- No badge, no star, no rank, no "best value" tag. The table describes; it does not judge.
- No colour-coded provenance — the source is a muted letter + legend, never a hue.

## Visuals — Read-Only GMP Display Contract (GMP-01, GMP-02)

The quietest surface in the app. Renders as the **last read block**, low on the page, immediately above the Q&A `2xl` divider (D4-02). Wrapped with `st.container(border=True)`. **No accent colour anywhere in this block.** Reads only from `data/gmp/<drhp_id>.json` (D4-03).

**Structure (new class `.drhp-gmp-block`), top to bottom:**
1. **Heading** (reuse `.drhp-snapshot-block-heading`): `Grey-market premium (unofficial)`. The word `GMP` in any prose carries the glossary tooltip.
2. **Persistent caveat** (`.drhp-gmp-caveat`, Small 12px muted, always visible — never collapsed): the one-line provenance/reliability warning (see Copywriting).
3. **Monochrome range strip** (`.drhp-gmp-range`): a `surface-secondary` track, `border-subtle` 1px, with **one muted `border-subtle`/`text-muted` tick per aggregator** positioned by value across the observed min–max span. Above/beside it, the range **headline** (Body 16, `text-primary`, `tabular-nums`): `₹38–₹52 across 3 sources` (₹ via `format_inr`). The visible spread between ticks is the honesty signal (D4-01). `role="img"`, `aria-label="Grey-market premium ranges from ₹38 to ₹52 across 3 sources; the sources disagree."`
4. **Per-source list** (`.drhp-gmp-source-line`, Small 12px muted): `Aggregator A ₹38 · Aggregator B ₹47 · Aggregator C ₹52 · as of {date}`. Each ₹ via `format_inr`.
5. **The disclosure** (inherited `st.expander`, unique `key`): header `What is GMP? Why we don't trust it`, default-collapsed, body = the genuine explanation (see Copywriting).
6. **Per-block disclaimer** (inherited `PER_ANSWER_FOOTER`).

**States:**
- **Default:** collapsed disclosure; caveat + range + sources visible.
- **No GMP reported:** `.drhp-not-disclosed` note `No grey-market premium is being reported for this IPO right now.` — never a fabricated number, never a zero.
- **Single source only (no spread):** show the single value + the caveat, and an explicit note `Only one source reported — no cross-source check available.` (absence of divergence is stated, not hidden).
- **Cache unreachable (error):** inherited amber `.drhp-refusal` style (NOT red) + infra copy + a `Retry` affordance.

**What NOT to do (anti-patterns):**
- No red/green; no accent; no up/down arrow; no "GMP suggests strong demand" or any signal framing (P21, D4-02).
- No single big authoritative number — the **range** is the headline, precisely because the sources disagree.
- No feeding GMP into any model surface — display-only, cache-only, imports no forecast module (D4-03).
- The block must not out-shout the peer table or red-flag table — it is typographically and chromatically the calmest block by design.

## Indian Formatting & Glossary Contract (UI-04)

### One shared `format_inr` utility (D4-07)

A single utility (recommended location `ui/format_inr.py`) is the ONLY place rupee amounts are formatted. It is retrofitted app-wide — the catalogue card, the metadata header, and the financials table all route through it.

> **FLAG-FORMAT (planner):** the existing `ui/catalogue.py::_format_issue_size` uses `f"₹{n:,} cr"`, which produces **Western** grouping (`1,234,567`). This is a latent bug. `format_inr` must implement **Indian** grouping (`12,34,567`), and `_format_issue_size` (plus every other ad-hoc ₹ render) must be refactored to call it. Do not re-implement grouping per surface (D4-07 = ONE utility).

**Contract:**
- **Input:** a rupee amount (in rupees). **Output:** a display string.
- **Indian digit grouping:** last three digits, then groups of two (`₹1,23,456`).
- **Auto-scale by magnitude:** `≥ ₹1,00,00,000` → `₹{x/1e7} crore`; `≥ ₹1,00,000` → `₹{x/1e5} lakh`; else grouped rupees (`₹45,600`). Scaled values show up to 2 decimals, trailing zeros trimmed (`₹1,247 crore`, `₹12.5 lakh`).
- **Negatives:** parentheses, same colour (inherited) — `(₹1,234 cr)`.
- **None / missing:** returns the em-dash sentinel `—` (renderers attach `aria-label="Not disclosed"`).
- The utility returns a string only; `tabular-nums` alignment is applied by CSS at render.
- Ratios (P/E, P/B, EV/EBITDA) and percentages (ROE) do **not** go through `format_inr` — they use a plain one-decimal numeric format with `tabular-nums`.

### Glossary tooltips (D4-08)

Eight terms carry the R-1 CSS-only tooltip: **RPT, QIB, NII, RII, GMP, OFS, DRHP, anchor investor.** Definitions live in `ui/copy.py` (scrubber-guarded, see Copywriting). Financial-ratio tooltips (P/E, P/B, EV/EBITDA, ROE) are **deferred** — added only later if trivial and non-cluttering.

## Cross-Cutting Invariants (re-stated, constrain every Phase 4 visual)

1. **No green/red coding, no badges, no verdict** — peer multiples and GMP are factual numbers only; no "cheap/expensive/high" colour; a low and a high multiple render identically (D4-09, P21).
2. **Honest sourcing, never a silent gap** — every peer cell either shows a value with its per-cell provenance marker, or an explicit `—` / `NM`; never a fabricated or interpolated number (D4-05).
3. **Absence shown as absence** — no DRHP peers → honest empty-state; no GMP → honest note; never a fabricated peer set or invented premium (D4-06).
4. **GMP is display-only and isolated** — monochrome, de-emphasised, low, behind a disclosure, cache-only, imports no model module; the cross-source spread is the honesty signal, never a demand signal (D4-01/02/03).
5. **One shared `format_inr`** — Indian grouping + lakh/crore + `tabular-nums`, retrofitted app-wide; the existing Western-grouping formatter is fixed, not duplicated (D4-07).
6. **Glossary is pure CSS, keyboard- and mobile-accessible** — no JS, no third-party tooltip lib, no hover-only `title` (D4-08, R-1).
7. **Extend, don't restyle** — no inherited token, type size, colour, or component CSS is redefined; Phase 4 adds new classes consuming the inherited `--drhp-*` properties.
8. **Streamlit container rule** — the peer block and GMP block use `st.container(border=True)` / self-contained single-call HTML; never a split-div wrapper (white-bar defect); every expander has a unique `key`.
9. **Scrubber covers all Phase 4 copy** — glossary definitions, GMP caveats, empty/error states land in `ui/copy.py` under the import-time banned-token assertion; the `sell` stem is avoided in the OFS/GMP wording (L8).
