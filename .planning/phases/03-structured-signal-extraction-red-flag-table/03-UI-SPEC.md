# Phase 3 — UI Design Contract: Structured Signal Extraction (Red-Flag Table)

> Visual and interaction contract for Phase 3's three NEW surfaces — the 7-field red-flag signal table, the IDF-ranked risk list, and the "show your work" methodology pane — generated by gsd-ui-researcher; consumed by gsd-planner, gsd-executor, gsd-ui-checker, gsd-ui-auditor. This contract **EXTENDS** the Phase 1 (`01-UI-SPEC.md`) and Phase 2 (`02-UI-SPEC.md`) design system. It does NOT restyle anything. Every Phase 1/2 token, color, type role, spacing value, and component is inherited unchanged; this document specifies only the deltas Phase 3 adds.

---

## Scope of This Contract

Phase 3 ships **three NEW user-visible surfaces**, all rendered on the existing snapshot page (`pages/02_snapshot.py`) and the existing Q&A chat surface:

1. **Red-flag signal table** — 7 fields (RPT % of revenue, OFS vs fresh-issue %, promoter pledge %, customer concentration, auditor history, debt trajectory, going-concern), each with a neutral high/med/low confidence label (EXTRACT-01, EXTRACT-02).
2. **IDF-ranked risk list** — a single ranked list ordered by issuer-specificity, each item carrying a neutral specificity indicator (P12 mitigation / ROADMAP SC-5).
3. **Methodology pane ("show your work")** — a one-click expansion on Q&A answers AND red-flag table fields, revealing retrieval query / retrieved chunks + scores / prompt / sources cited / committed eval scores (METHOD-01, D3-16, D3-17).

Everything else on the snapshot page (the 6 Phase 2 field blocks, catalogue cards, citation chips, disclaimer surfaces, Q&A chat) is **already shipped and unchanged**.

## Discretion Resolved (the three genuinely-open questions)

`03-CONTEXT.md §Claude's Discretion` explicitly deferred three visual/interaction decisions to this UI-SPEC. They are resolved here, prescriptively:

### R-1 — Red-flag table layout: **definition-list "field rows," NOT a dense data grid, NOT cards.**

A **stacked field-row layout** (one row per field: field label on the left, value + confidence label + methodology affordance on the right), rendered inside a single `.drhp-redflag-table` container that visually matches the existing `.drhp-snapshot-block` chrome (white bg, `border-subtle`, `8px` radius, `lg` padding).

**Why this over the alternatives:**
- A **dense HTML data grid** breaks on the 375px mobile-first viewport (Indian retail is mobile-first, UI-01) — 7 fields × (value + confidence + expand) is too many columns. The Phase 2 financials table already needed `overflow-x: auto`; replicating that for the headline red-flag surface would bury the signal behind horizontal scroll.
- **Cards** (one card per field) waste vertical space and visually imply equal weight / a dashboard "metric tile" aesthetic that drifts toward the badge/scorecard look L3-1 forbids.
- **Field rows** degrade gracefully: on desktop the value sits inline-right of the label; on mobile it stacks below the label. Same pattern as the Phase 1 DRHP metadata header label-value pairs (proven responsive).

The 7 fields render in a **fixed canonical order** (locked so the auditor can diff field-by-field across IPOs):
1. RPT % of revenue
2. OFS vs fresh-issue %
3. Promoter pledge %
4. Customer concentration
5. Auditor history
6. Debt trajectory
7. Going-concern mentions

### R-2 — Specificity indicator rendering: **a neutral text rank label + a neutral grey specificity meter, NO color judgment.**

Each risk item in the IDF-ranked list shows:
- A **rank counter** in the same muted style as the Phase 2 `.drhp-risk-item-counter` ("Risk N of M"), extended with a neutral specificity word: `Risk 1 of 7 · Issuer-specific` / `… · Mostly issuer-specific` / `… · Industry-standard`.
- A **monochrome specificity meter** — a thin horizontal bar (`.drhp-spec-meter`) filled in **accent slate-indigo `#1E40AF`** proportional to the normalized IDF score, on a `surface-secondary #F4F5F7` track. This reuses the EXACT visual grammar of the Phase 2 OFS split bar (accent fill on neutral track) — NOT a new color, NOT red/amber/green. Higher fill = more issuer-specific = foregrounded by rank. A text percentage / band label always accompanies the meter so color is never the sole differentiator (WCAG 1.4.1).

The specificity word maps to IDF bands (thresholds tuned empirically by the planner per D3-14, documented in the labeling rubric): top band → `Issuer-specific`, middle → `Mostly issuer-specific`, bottom → `Industry-standard`. Accent fill is a **specificity indicator, not a quality verdict** — an "Industry-standard" risk is neither good nor bad, just common.

### R-3 — Methodology-pane expansion affordance: **a reused `st.expander` with a neutral "Show your work" label, opening inline below the claim/field — NOT a modal, NOT a sidebar.**

The pane is the SAME `st.expander` primitive the Phase 1 citation expanders already use — keeping one consistent "click to reveal source detail" mental model across the whole app. Its trigger label is the literal copy `Show your work` (no emoji, no icon). It opens inline directly beneath the answer / field row it explains, default-collapsed. This:
- reuses `ui/expander.py`'s descriptor pattern and the `.drhp-snippet` / `.drhp-snippet-metadata` CSS already in `drhplens.css`;
- avoids a modal (which would steal focus from the snapshot the user is reading) and a sidebar (which Phase 1/2 deliberately suppress for a single-column mobile-first app);
- nests cleanly: a red-flag field row owns its citation expander(s) AND one "Show your work" expander, in that order.

## Visuals — Red-Flag Signal Table Contract

The headline Phase 3 surface. Renders inside a single block matching `.drhp-snapshot-block` chrome, positioned on `pages/02_snapshot.py` **directly after the Phase 2 metadata header and before (or alongside) the 6 Phase 2 field blocks** — planner picks exact ordinal, but it must be above-the-fold-adjacent on mobile since it is the Phase 3 headline.

**Structure (new class `.drhp-redflag-table`):**
- Block heading (Heading 20/600, reuses `.drhp-snapshot-block-heading`): `Red-flag signals`
- Sub-line (Small 12 muted): the red-flag sub-line copy above
- **7 field rows** in canonical order, each a `.drhp-redflag-row`:
  - **Field label** (Body 16/600 muted, left): e.g. `RPT % of revenue`
  - **Field value** (Body 16/400 primary, right on desktop / below on mobile): cited prose with inline `[n]` citation chips rendered by the UNCHANGED `ui/chip.py` `render_answer_with_chips()` — every value carries `claim_id` citations exactly like a Q&A answer (D3-17, reuse of grounded pipeline). Numeric values use `tabular-nums`.
  - **Confidence label** (Small 12 muted, beneath/after the value): `Confidence: high|medium|low` — text only, no pill, no color (L3-2).
  - **Citation expander(s)** (inherited `st.expander`, `.drhp-snippet`): one per deduped chip, exactly as Phase 2 field blocks.
  - **"Show your work" expander** (see Methodology Pane Contract): default-collapsed, below the citation expanders.
- A `1px` `border-subtle` separator between rows (not zebra fill, to stay calm and avoid implying alternating significance).

**Not-disclosed rows (L3-3):** when a field's `SnapshotRecord` value is a `RefusalResponse`, the row renders the field label + the inherited `.drhp-not-disclosed` italic note `Not disclosed in DRHP` in place of the value. The confidence label is OMITTED entirely (absence is not "low confidence" — D3-03). The row still renders — never hidden.

**Numeric-gate-blocked values (L3-9):** if `cite_check`'s numeric sub-check fails for an emitted number, the planner treats it as not-disclosed for rendering purposes and uses the blocked-copy string — never a value without a grounding citation.

**What NOT to do (anti-patterns):**
- No red/amber/green on any value, label, or row. A 60%-pledge promoter and a 0%-pledge promoter render in identical chrome.
- No `🚩` / `⚠️` / severity icon — the surface is named "red-flag" but renders zero red and zero flags (the honesty is in surfacing the factual value, not in coloring a judgment).
- No "score out of 10," no aggregate "risk score," no badge. Each field stands alone as a cited fact.
- No confidence pill with a fill color. Confidence is muted text.

## Visuals — Methodology Pane ("Show Your Work") Contract

METHOD-01's first demoable surface. Wired on Q&A answers AND red-flag fields (L3-5). Cached content only (L3-6).

**Affordance:** an inherited `st.expander` with header copy `Show your work`, default-collapsed, rendered inline below the claim/field's citation expanders. No modal, no sidebar (R-3).

**Pane content (top to bottom), all from cached `claim_id` data + the latest committed eval report — NO live LLM call:**
1. **Retrieval query** (label Small muted `Retrieval query` + Body 16 mono-ish primary): the exact query string that retrieved evidence for this claim/field, from cached trace.
2. **Retrieved chunks (with scores)** (label + a list): each chunk shows its rerank/retrieval score (Small muted, `tabular-nums`) + a truncated chunk preview in `.drhp-snippet` style (italic, `border-subtle` left rule). Reuses the snippet styling already in CSS.
3. **Prompt used** (label + a `surface-secondary`-paneled, scrollable, max-height block): the prompt template for this claim/field, from cached trace.
4. **Sources cited** (label + the same `claim_id · DRHP page N · section` metadata footer already produced by `ui/expander.py`'s `metadata_footer`): reuse verbatim.
5. **Eval scores** (label `Eval scores (from the latest committed report)` + Small muted lines): faithfulness + citation-accuracy at answer/field level, plus the **numeric confidence score** `0.00–1.00` and its rubric tier (this is the ONLY place the numeric confidence surfaces — L3-2). Followed by the eval-provenance note copy (clarifies these are committed, not live — D3-17).

**States:**
- **Default:** collapsed; header `Show your work` in inherited expander chrome.
- **Expanded:** content above; focus ring on the header is the inherited 2px accent ring.
- **Eval score not yet committed:** show the eval-not-yet-available copy in place of the eval block; the rest of the pane (query/chunks/prompt/sources) still renders from cached trace.
- **Reduced motion:** expander opens instantly (inherited `prefers-reduced-motion` rule already in CSS).

**What NOT to do:**
- No live per-expand LLM call or live eval recompute (L3-6) — purely cached/committed data.
- No fake eval numbers — if no committed score exists, say so (mirrors the Phase 1 `/methodology` "no fake DS rigor" rule).
- No color-coding of eval scores (a 0.91 faithfulness is not green). Muted text only.
- The pane never exposes a banned token from the prompt template — the scrubber covers methodology copy too (L3-8); if a cached prompt string would surface a banned token to the user, it is rendered as-is ONLY because it's developer-authored prompt text, but any NEW user-facing label in the pane is scrubber-guarded via `ui/copy.py`.

## Cross-Cutting Invariants (re-stated, constrain every Phase 3 visual)

1. **No green/red coding, no badges, no severity icons** — confidence labels, specificity words, field values, and the IDF meter are all monochrome/muted; accent is magnitude, never verdict (L3-1).
2. **Span-level citations only** — every red-flag value and risk item renders from the `claim_id` structure via the unchanged `ui/chip.py`; the methodology pane consumes the same cached `claim_id` data (L3-5/6).
3. **Numbers are grounded or not shown** — the numeric-faithfulness gate blocks any unsourced number; a blocked field renders as not-disclosed/blocked copy, never an unsourced value (L3-9).
4. **Absence is an honest signal** — "Not disclosed in DRHP" rows always render; never hidden, never conflated with low confidence (L3-3).
5. **No fake DS rigor** — the methodology pane shows committed eval scores or explicitly says none exist yet; no placeholder numbers (D3-17).
6. **Cached, not live** — the methodology pane never triggers an LLM call or eval recompute on expand (L3-6).
7. **Mobile-first** — every contract above states mobile behaviour first; the red-flag table is field-rows precisely so it survives 375px without horizontal scroll.
8. **Scrubber covers all Phase 3 copy** — new user-facing strings land in `ui/copy.py` under the import-time banned-token assertion (L3-8).
