<objective>
Build the offline red-flag extraction pipeline — the data layer behind the Phase 3
headline surface. Mirror `pipelines/snapshot.py` exactly: run the EXISTING compiled
agent graph over the 7 canned `REDFLAG_QUERIES`, store each field as a cited
`GroundedAnswer` (with a deterministic confidence tier) or an honest
`RefusalResponse`, block any field whose number fails the Plan 02 grounding gate,
reuse the snapshot's `ofs_fresh` for field #2, and cache the result to
`data/redflag/<drhp_id>.json` (EXTRACT-01/02). Plus the only genuinely-new
algorithm in the phase: `pipelines/risk_idf.py`, a ~40-line stdlib in-corpus IDF
ranker with a hand-curated boilerplate floor that foregrounds issuer-specific
risks (P12 / D3-14).

Purpose: This is the "storage is the integration bus" write side — batch
precompute that the Plan 04 UI reads with zero request-time LLM calls.
Output: `pipelines/redflag.py`, `pipelines/risk_idf.py`,
`eval/gold/boilerplate_phrases.txt`, and two Wave 0 test modules flipped to green.
</objective>
